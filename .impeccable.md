## Design Context

### Users
- 主要用户是《Idle Champions of the Forgotten Realms》的活跃玩家，会在配队、查 seat 冲突、核对英雄技能 / 天赋 / 皮肤时打开这个站点。
- 这是一个偏个人决策辅助的资料站，核心场景是查资料、做限制筛选、编辑阵型、核对候选英雄与保存方案。
- 当前最核心的工作不是“看百科”，而是快速定位一个英雄的完整资料，并把这些信息转化为阵型判断。

### Brand Personality
- 理性、克制、战术化、可解释。
- 页面应该给人“资料很全，但不乱”的感觉，同时保留一点奇幻冒险游戏的金属感与徽记感。
- 情绪目标是降低信息焦虑、提升判断信心，而不是做成夸张炫技的展示页或通用 SaaS 控制台。

### Aesthetic Direction
- 默认沿用站点现有深色主题，以铜金色和冷钢蓝做少量点缀，营造夜间战术台 / 资料卷宗的观察窗感。
- 头像、标签和卡片允许有更强的层次与结构感，但不能大面积遮挡信息；重点是做出“有判断的强调”，而不是堆光效。
- 参考方向是密度高、分区清晰、方便跳读的数据页；不做平铺百科，也不做多层弹窗。
- 明确避免：模板化后台卡片墙、彩虹标签堆砌、为了“高级感”而牺牲扫描效率的装饰。

### Design Principles
1. 先给结论性摘要，再给完整细节，最后给原始字段。
2. 高密度信息必须按问题分区，让用户一眼知道去哪找。
3. 任何“所有信息”都不能以牺牲扫描效率为代价；原始字段默认收起，但必须可达。
4. 强调对象时优先使用角落舞台、徽记、偏移构图等结构手法，而不是直接放大挡住内容。
5. 公共样式保持克制，只有当前任务需要突出的页面才做更强表现，同时兼顾移动端与 GitHub Pages 静态站点环境。

## UI Audit

### Overall Signature
- 整站当前最有辨识度的不是单个卡片，而是“深色战术台 + 铜金 / 冷钢蓝点缀 + 卷宗式层级”的连续体验。
- `src/styles/foundations/tokens.css` 已经形成稳定的深色 OKLCH 调色板，符合 `impeccable` 对统一色彩重心、稀缺强调色和有色中性色的要求。
- 中文界面使用 `PingFang SC` + `Songti SC`，在信息密度和奇幻资料感之间取得了比常见 SaaS 字体栈更强的识别度。

### Shell And Navigation
- 站点头部不是普通仪表盘横条，而是悬浮战术壳层；压缩态、移动端菜单和语言切换共同组成“可收拢的指挥台”。
- 主导航适合继续保持“重结构、轻图标”的方向，避免退化成模板化图标标签墙。
- 头部与导航的装饰应继续来自边框、切角、局部高光、密度变化，不使用渐变文字、玻璃化泛滥或彩色侧条。
- 命中工作台路由时，桌面端头部应默认保持紧凑小导航状态；页面内返回、复制链接和局部动作全部下放到工作台工具条，不再塞回导航。

### Workbench Signature
- 当前桌面端页面签名已经稳定为“上方小导航 + 下方单一工作台壳体”；不要再回到“大页头 + 页面内多张并列大卡”的旧框架。
- 工作台外壳必须保持左右一体：左侧更轻、更像前景抽屉，右侧更深、更像资料画布；两侧顶部 chrome 读起来要像同一条工具带。
- 左栏页面的左工具条和左主体必须属于同一块抽屉；收起后左主体、边框和 gutter 一起退场，只留下紧凑锚点按钮。
- 动效优先“结构一起平移”而不是局部补跳；右区悬浮回顶按钮只在内部滚动后出现，不把整页滚动条重新带回来。

### Information Surfaces
- 通用表面系统的强项是层次稳定：hero、surface、detail-section、metric 都共享一套材质语言，但仍能通过内边框、台阶式标题和分区线区分层级。
- 结果卡当前是视觉表达最强的一层，尤其是立绘舞台、头像框和 spotlight 版式，已经具备 `impeccable` 所要求的“结构性强调”而不是模板化图块堆叠。
- 后续继续强化时，优先做信息编排和留白节奏，不要把所有卡片都推向更重的装饰。

### Page-Level Character
- 英雄详情页适合继续保持“摘要 -> 分区 -> 原始字段”的卷宗路线；这是整站最接近产品核心任务的页面。
- 筛选页、宠物页、立绘页和阵型页应继续共享结果卡与侧栏契约，减少视觉跳变，让用户把认知成本留给业务判断而不是页面适应。
- 阵型页的移动端编辑器是当前最值得保持的特征之一：它把复杂棋盘操作压缩成单点聚焦的移动流程，而不是删功能保布局。

### Anti-Slop Guardrails
- 禁止回到默认后台审美：统一卡片墙、巨型彩色指标、圆角图标配标题、重复 hero metric 模板。
- 禁止使用渐变文字、粗侧边色条、无意义玻璃化、装饰性 sparklines。
- 强调靠构图、版式、边界和材质变化完成；字体、颜色和动效只做辅助，不做主角。

### Next Visual Direction
- 如果后续继续打磨视觉，优先级应是：详情页分区跳读效率、结果卡多态一致性、头部压缩态稳定性、移动端阵型操作节奏。
- 允许在关键页面继续做更有记忆点的舞台式构图，但必须始终让扫描效率高于装饰冲动。
