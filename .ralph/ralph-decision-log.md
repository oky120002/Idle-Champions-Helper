# Ralph 决策记录

- 决策点：英雄筛选结果卡如何在不脱离现有视觉语言的前提下压缩高度并改善立绘展示
  选择方案：采用“左侧信息摘要 + 右侧头像舞台 + 顶部立绘背板”的横向卡片结构，并把属性概览改成更紧凑的两列摘要面板
  推荐理由：能在保留现有金属感与卡片层次的同时减少纵向堆叠，让名字、定位、联动队伍和属性标签更快扫读，也避免立绘被整卡背景拉伸后只剩局部

- 决策点：英雄页随机排序是否写入 URL 或筛选状态
  选择方案：沿用立绘页和宠物页的本地随机种子做法，只影响当前过滤结果与展示窗口，不写入 URL
  推荐理由：交互和已有两个随机排序入口保持一致，用户可以随时重洗当前结果，但不会污染分享链接或破坏已有筛选恢复逻辑
