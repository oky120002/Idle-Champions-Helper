# 页面标题与统计条规范

- 日期：2026-04-18
- 作用：约束各顶层页面的标题区块、简介文案和小型统计条，避免每页各做一套，且保证后续 AI 修改时只需读取这一篇叶子文档。

## 适用范围

- 使用 `SurfaceCard` 作为页面首块容器的顶层功能页。
- 页面首屏需要同时表达“这页是干嘛的”和“当前数据规模 / 匹配规模”的场景。

## 布局规则

1. 标题、说明与统计条共用同一个 header 容器；桌面端优先左右并排，移动端自动折成上下结构。
2. 标题文案保留一句主标题；只有在真的能帮助理解页面范围时才保留 eyebrow。
3. 描述文案只保留一段短说明；如果标题已经足够明确，可以不写描述。
4. 统计条使用 3-6 个紧凑 metric，放在标题右侧；不要再在标题下方重复放一整排大卡片。
5. metric 必须优先放用户当前最需要扫到的数量，例如“当前匹配”应靠前，历史性或背景性数据靠后。

## 视觉规则

- metric 采用小型胶囊 / 微卡片，不抢标题层级；标签字号应明显小于数值。
- 统计条允许自动换行，但每个 metric 应保持固定内边距和统一圆角。
- 页面首块应优先节省纵向空间；标题区块不要和正文之间堆过厚的留白。
- 同一页面里，如果标题区块已提供统计条，正文中不要再复制同一组统计卡。

## 文案规则

- 标题与说明默认跟随当前界面语言。
- 卡片内若需要双语同时出现，优先只给“主名称”做双语并排展示，其它辅助字段只跟随当前语言。
- 统计条标签保持短词，不写完整句。

## 当前项目落地

- `src/components/SurfaceCard.tsx` 提供 `headerAside`，用于承载复用统计条。
- 复用统计条组件放在 `src/components/PageHeaderMetrics.tsx`。
- 当前已落地页面：`src/pages/ChampionsPage.tsx`、`src/pages/IllustrationsPage.tsx`、`src/pages/PetsPage.tsx`、`src/pages/VariantsPage.tsx`、`src/pages/PresetsPage.tsx`。

## 修改顺序

1. 先改 `src/components/SurfaceCard.tsx` 与 `src/styles/shared/surfaces.css`。
2. 再改具体页面的 metrics 组件与页面入口。
3. 最后确认正文里没有残留的重复统计块。
