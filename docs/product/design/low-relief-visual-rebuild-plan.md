# 整站低浮雕扁平化重构计划

- 日期：2026-04-22
- 状态：执行中
- 目标：把整站从“深色战术台 + 明显材质层次”重构为“低浮雕、低材质、低动效”的统一视觉系统；按未上线产品处理，不保留旧视觉兼容路径。

## 总原则

1. 新视觉基线是唯一事实源，不做新旧双轨并存。
2. 共享层先重写，再收页面私有样式；不在旧材质规则上继续打补丁。
3. 默认收平所有壳层、控件和辅助信息区，只让真正需要聚焦的内容区保留少量深度。
4. 不保留 legacy / compat / deprecated 样式分支、旧类名兜底或双重断言。

## 范围与取舍

### 统一收平

- 顶部壳层、主导航、语言切换、背景光晕
- 通用 panel、section、metric、标题区块
- 筛选抽屉、chips、badge、表单、按钮
- workbench chrome、结果头部、滚动辅助控件
- 阵型布局库、棋盘外壳、移动端编辑器外壳
- 立绘页、宠物页、变体页、筛选页的页面级辅助面板

### 保留少量深度

- 结果卡
- 英雄详情页的 dossier / media 主区

### 例外区的限制

- 只保留 1 层主舞台背景和 1 层轻阴影
- 删除厚重发光、强内凹、重高光、明显浮雕边
- 例外区必须继续承担“内容聚焦”职责，而不是单纯装饰

## 实施顺序

1. foundations / shared 基线重建
   - 收窄圆角体系
   - 收弱阴影体系
   - 减少渐变层和 inset 高光
   - 收敛 hover / active / focus 反馈
2. app 壳层统一
   - 顶部导航、语言切换、sticky header、背景层一起收平
3. workbench / filters / controls / formation 共享层统一
   - 删除重复材质写法，统一成新的低浮雕密度
4. results / champion-detail 例外区重做
   - 保留少量聚焦层次，但同步瘦身
5. 页面私有样式清理
   - 立绘、宠物、变体等页面只保留编排和少量例外，不再重复实现共享材质
6. 文档与测试同步
   - 视觉规范、测试断言、回归基线一起切换到新系统

## 共享层改造要求

### foundations

- 允许直接调整 token，建立新的阴影、边框、圆角和交互强度基线
- 不保留旧 token 仅为兼容服务

### surfaces

- panel、metric、section 从“多层材质卡片”改为“低浮雕面板”
- 删除无必要的重投影和重复高光

### filters / controls

- 所有 chip、button、badge、input 默认更薄、更平、更克制
- hover 以轻微提亮或边框变化为主，不依赖厚重抬升

### workbench / formation

- chrome、抽屉、布局库和棋盘外壳统一收平
- 结构识别依赖分区、边界和留白，而不是强材质堆叠

## 动效基线

- 采用“明显收敛”策略
- hover 位移控制在极小幅度
- 去掉依赖阴影跳变制造层次的反馈
- 保留结构型动效：抽屉开合、header 收紧、结果展开/收起、必要的滚动辅助

## 文档与测试要求

- 文档只描述新视觉系统，不保留旧风格说明
- 测试断言只接受新基线，不允许“旧样式或新样式都算过”
- 必跑：`lint`、`typecheck`、`build`
- 关键回归：导航、header 收紧态、workbench 一体结构、筛选区、结果卡、详情页主区、阵型页
