# 移动端兼容性开发规范

- 生效日期：2026-04-14
- 核心结论：移动端默认禁止横向滑动作为主要交互；尤其不能把主导航、筛选、按钮组或关键信息列表做成必须左右拖动才能正常使用的形态。

## 适用范围

- 顶部主导航
- 筛选区与筛选结果快捷入口
- 标签组、按钮组、切换组
- 主要信息卡列表
- 首屏或长页面里的 sticky 区域

## 强制规则

1. 默认不得出现“为了访问主要功能而必须横向滑动”的交互。
2. 默认不得出现页面级横向溢出；验收时应确认 `document.documentElement.scrollWidth <= document.documentElement.clientWidth`。
3. 空间不足时，优先改为折叠面板、抽屉、分步展开、纵向分组、两列网格或更少但更明确的常驻入口。
4. sticky 区域要先压缩常驻信息量；不能把完整桌面导航、长说明和大段副标题一起长期固定在顶部。

## 默认替代顺序

1. 先减少常驻内容。
2. 再折叠低频入口。
3. 再把并列入口改成纵向分组或两列网格。
4. 最后才考虑更深层级导航。

默认不采用的方案：横向滚动导航、横向滑动筛选条、必须左右拖动才能看全的按钮组、依赖用户自行发现“还能横滑”的隐藏式交互。

## 例外边界

只有内容天然属于横向画布时，才允许讨论横向交互，例如阵型画布、时间轴、大表格、以横向浏览为核心的媒体内容；即便如此，也要同时满足：

- 用户明确认可
- 页面提供非横向滑动的替代入口或摘要
- 不影响主任务闭环

## 当前项目落地

- 2026-04-14 已废弃“一级导航单行横向滑动”方案，改为紧凑头部 + 折叠式主导航面板 + 纵向网格入口。
- 阵型编辑和数据密集页面允许存在宽视觉区，但主流程仍必须能在非横滑路径下完成。
