# 滚动稳定性：问题与修复方案

- 日期：2026-04-13
- 目标：沉淀英雄筛选页滚动跳动的复现结论与当前修复方案。

## 复现结论

本次针对 `#/champions` 做了浏览器级复现，确认存在两类独立但会叠加的体验问题：

1. 点击筛选条件时，浏览器根滚动锚定会尝试“帮忙”维持视口位置，导致页面出现突兀的上下跳动。
2. 当原始列表很长，而筛选结果骤减到很少甚至为空时，页面总高度会突然缩短；如果当前窗口滚动位置已经低于新页面允许的最大值，浏览器会立刻把整页 `scrollY` 夹回更靠上的位置。

第二类问题即使在禁用根滚动锚定后仍会出现，所以不能只靠 `overflow-anchor: none` 解决。

## 当前修复方案

### 全局层：禁用根滚动锚定

在 `src/styles/global.css` 中对 `html, body` 设置 `overflow-anchor: none;`，去掉浏览器对根页面的自动滚动补偿。

### 页面层：结果区高度过渡，而不是固定小窗

`src/pages/ChampionsPage.tsx` 中把结果区域改为自然高度的 `results-panel-shell` / `results-panel` 结构，并在 `src/styles/global.css` 中维护 `results-panel-shell`、`results-panel`、`results-panel__meta`、`results-grid--stable` 等样式。

目的：保持页面仍然是正常整页滚动，不牺牲大屏 / 小屏下的卡片可见数量；当结果从很多条骤减到很少时，先临时保留旧高度，再过渡到新高度。

### 结果骤减时必要才平滑带回结果区

当用户已经滚到结果区较深位置，并且筛选后结果明显变少时，会平滑滚动到结果区顶部附近，而不是被浏览器一下子拉到页面更靠上的随机位置。

### 桌面端筛选区改为粘性侧栏

在桌面宽度下改为“两栏工作区”：左侧筛选区、右侧结果区，左侧保持 `sticky`。这能减少视线来回跳，也保留结果区自然高度。

### 其他补充稳定措施

- 首次出现“当前筛选”摘要时预留占位，避免第一条筛选一出现就把结果说明整体向下推一行。
- 结果默认先展示 `48` 名，并提供“显示全部 / 收起”切换。
- 空结果时补“放开筛选 / 一键清空”快捷操作。
- 增加结果区 `顶部 / 到底` 快捷按钮，并放宽桌面端页面宽度。

## 当前仍有效的判断

- 这个问题本质上是“浏览器滚动行为 + 页面高度突变 + 结果区布局策略”的组合问题。
- 单纯修补按钮点击逻辑，或者在事件后强行 `window.scrollTo(...)`，都只是临时补丁。
- “固定结果滚动小窗”虽然能止住跳动，但会牺牲可见卡片数量，不适合这个筛选条件很多的页面。
