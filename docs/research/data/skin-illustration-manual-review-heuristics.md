## 1. 当前已经稳定下来的复核原则

### 1.1 什么时候“值得写 override”

目前真正落地为 override 的样本，基本都满足下面至少一条：

- **替代 pose 明显修复了“主体过小 / 画布过空”的问题**
  - 例：`133` 星运伊芙琳
- **替代 pose 明显收回了默认 frame 里横向外展过大的武器 / 肢体，主体更集中**
  - 例：`473` 飞升阿斯代伦
- **替代 pose 把默认图里明显偏窄或异常展开的构图，拉回到同英雄组更自然的范围**
  - 例：`368` 半精灵变异哨兵
  - 例：`520` 舞厅蔚

换句话说，真正值得写 override 的，不是“有一点不一样”，而是：

- 替代图在肉眼上有**明确且稳定的优势**
- 这种优势不只是“更大一点 / 更小一点”，而是**整体更像正式主立绘**

### 1.2 什么时候“不值得写 override”

目前被明确判为“不建议写 override”的样本，常见于下面几类：

- **替代 pose 只有轻微尺寸变化，没有明显更自然**
  - 例：`362`、`290`、`515`、`111`、`452`、`550`
- **替代 pose 虽然更紧凑，但引入了更严重的连接错误**
  - 例：`123` 星运弗里，手部与法杖断开，已撤回 override
- **detached 道具 / companion / 浮空组件在多候选 pose 里都持续存在**
  - 例：`282`、`298`、`306`、`351`、`496`
- **主题本身就是小体型 / 玩偶 / 毛绒 / 宝宝**
  - 例：`470`、`471`、`409`
- **主题本身就天然更宽 / 更高 / 更复杂**
  - 例：`367` 半人马、`210` 玻璃罩、多主体包装类主题

## 2. 当前最有用的判断经验

### 2.1 “detached 不等于错”

这是目前最重要的一条经验。

很多高风险样本会被系统标成：

- 浮空武器
- 浮空书
- 浮空眼球
- companion / 特效独立成块

但人工复核后要先问的不是“它 detached 了没有”，而是：

- 这个 detached 元素是不是该主题的一部分？
- 它在多个候选 pose 里是否都持续存在？

如果答案是“是，而且一直都在”，那更可能是：

- **主题构图本意**

而不是：

- **当前 frame 选错**

### 2.2 “更紧凑”不等于“更对”

之前已经出现过误判：

- `123` 星运弗里

候选图从尺寸与收敛度上看更像“更漂亮”，但真正肉眼看细节时：

- 手和法杖是断开的

这说明：

- 不能只因为候选更紧凑、fill 更高、面积更像组内中位数，就直接录 override
- **连接关系与肢体完整性永远比尺寸分数更重要**

### 2.3 “同主题组持续出现的 detached 元素”优先判成主题本意

这条经验在 alpha 审计后更明显：

- `282` 女巫之光 BBEG：漂浮眼 / 漂浮书
- `298` 假期地下城主：漂浮饮料
- `306` 骄傲父亲梅亨：酒壶 + 蝙蝠
- `496` 塞伦涅守望者：月牙 / 星星 / companion 感很强

如果某个 detached 元素：

1. 在当前图里存在
2. 在多个高排名候选 pose 里也存在
3. 且与皮肤主题非常一致

那它大概率应该先归类为：

- **主题道具**

而不是：

- **浮空 bug**

### 2.4 “当前 pose 错了”通常会有更强对照信号

真正的错误 frame，往往会出现下面这些更强信号之一：

- 替代 pose 明显修复断手 / 断杖 / 断肢
- 替代 pose 明显减少莫名外展
- 替代 pose 明显把人物主体拉回画布中心
- 替代 pose 在同一 sequence 的相邻 frame 中也持续更优，不是单帧碰巧好看

如果这些信号都没有，只是：

- 稍微大一点
- 稍微收一点
- 稍微换个脚步

通常不值得写 override。

## 3. 当前推荐的人工复核流程

### 3.1 先看“为什么被筛出来”

先区分候选来自哪类审计：

- 尺寸审计
  - 更适合抓主体过宽、过高、过小
- alpha 连通域审计
  - 更适合抓 detached prop、多主体构图、内部留白

不同来源，看的重点不一样：

- 尺寸审计更看主体占比
- alpha 审计更看 detached 元素是不是主题本意

### 3.2 先看 current，再看 top 候选 8 张

当前最好用的方式还是：

1. 渲染 `current + top 8 candidate poses`
2. 先看是否存在“压倒性更对”的候选
3. 再看这个候选有没有细节副作用

### 3.3 不确定时，要看“相邻 frame 是否同样成立”

如果一个候选只在单帧看着更好，但：

- 前后相邻 frame 并不稳定
- 或只是碰巧避开了某个 detached 道具

那不要急着写 override。

### 3.4 最终裁决优先级

当前建议按这个优先级做判断：

1. 肢体 / 武器 / 手部连接是否完整
2. 主体是否清楚、稳定、像正式主立绘
3. detached 元素是否属于主题本意
4. 尺寸、fill、面积是否更接近组内中位数

不要把第 4 条放到前面。

## 5. 当前经验沉淀后的直接结果

截至 2026-04-16，这些经验已经能支撑两个直接动作：

- 第一轮尺寸审计候选池已经基本跑空
- 第一轮 alpha 审计候选池在补完 top 样本人工复核后，也已经跑空

也就是说，后续如果还要继续系统扩样，不能只重复：

- 看宽高面积偏差
- 看 detached 连通域数量

下一步更应该补的是：

- **current vs candidate 的 delta 审计**

也就是比较：

- 候选 pose 是否真的比 current 更连贯
- 候选 pose 是否真的减少 detached area
- 候选 pose 是否只是把主题 props 暂时移开了一点

## 6. 后续追加格式建议

以后每次新增人工结论，建议按下面格式往本文追加：

```md
## 2026-xx-xx 第 N 轮补充经验

- 新增样本：
  - `skinId`：一句话结论
- 新增可复用规则：
  - 规则 1
  - 规则 2
- 被推翻 / 修正的旧经验：
  - 旧判断 -> 新判断
```

这样这份文档才能真正“积少成多”，而不是每轮重写一遍。
