# 全站页面工作台设计规范

- 日期：2026-04-22
- 作用：固定当前已落地的 `PageWorkbenchShell` 视觉签名、结构语义、滚动模型和关键动效，作为后续细节迭代与跨页复核的设计基线。
- 范围：这里只展开“现在已经实现成什么样”；页面归位、推广范围和旧实现清退口径继续以 `docs/modules/shared-components/filter-workbench-rollout-plan.md` 为准。

## 结构签名

- 桌面端统一使用“双层页面”：
  - 上层是默认紧凑态的全站小导航，像悬浮指挥条，而不是大页头。
  - 下层是单一外层工作台壳体，保留站点背景与光斑，不再把主页面拆成多张彼此独立的大卡。
- 工作台内部统一保留“左语义 / 右语义”：
  - 有左栏页面：左侧是抽屉式 pane，右侧是主内容画布。
  - 无左栏页面：只隐藏左 pane，本体仍走同一套右区结构，不再额外维护单栏页面体系。
- 移动端继续退化为普通单列网页滚动，不强行维持桌面高度锁定和双栏内滚。

## 顶部小导航

- 命中工作台路由时，桌面端头部默认直接进入 condensed 态，不再等待整页滚动后压缩。
- 小导航职责只保留全站切页、当前工作区识别和低频控制，不再承载页面内返回入口、筛选动作或详情页局部操作。
- 视觉上保持“重结构、轻图标”：
  - 外层是悬浮战术壳；
  - 导航项是横向标签式切页；
  - 装饰来自切角、边缘光、密度变化，不用渐变文字、彩色侧条和泛滥玻璃化。

## 工作台壳层

- 外层工作台必须是一整块连续材质：圆角比站点头部更小、更克制，边缘更像战术台舱门，不像独立弹窗。
- 左右 pane 都挂在同一个外壳里，不允许再次拆成两个互不相干的大卡。
- 左 pane 材质更轻、更透，像前景抽屉；右 pane 更深、更实，像主内容画布。
- 左右顶部 chrome 高度必须一致，并作为同一条工具带来阅读，不能出现“左边一条、右边再一条”的断裂感。

## 工具条语义

- 工作台工具条统一拆成三段：
  - `toolbarLead`：页面标记、返回入口或左侧识别信息；
  - `toolbarPrimary`：页面主标题、kicker、上下文说明；
  - `toolbarActions`：badge、复制当前链接、稳定动作。
- 这三段的稳定展示骨架统一由 `src/components/workbench/WorkbenchScaffold.tsx` 承接；页面只负责传业务文案、统计值和动作回调。
- 有左栏时：
  - `toolbarLead` 留在左 pane 顶部；
  - `toolbarPrimary + toolbarActions` 留在右 pane 顶部；
  - 两段需要保持同高、同底色体系和连续边界，像一条被中缝切开的工作带。
- 无左栏时：
  - `toolbarLead` 内联并入右侧工具条左端；
  - 右端动作仍固定靠右；
  - 不渲染空的左轨、占位 gutter 或残余边框。

## 左侧状态

- `展开态`
  - 左工具条和左主体必须是一体化抽屉，不是上面单独一条、下面再接筛选区。
  - 中间分隔只是一条细边界，强调左右职责，不制造裂缝感。
- `收起态`
  - 左抽屉主体、边框、阴影和 gutter 全部消失；
  - 只保留一个贴在左上角的紧凑锚点按钮；
  - 收起后右区立即补满，不留残余窄轨。
- `无左栏态`
  - 不渲染 toggle；
  - 不渲染左 pane；
  - 仍保留同一套右区 chrome、内容头和主滚动模型。

## 动效与过渡

- 抽屉动效目标是“侧层平移”而不是“左边先滑完、右边再补跳一下”。
- 左 pane、右 pane 和中缝必须同时参与位移；用户应感知为同一块工作台结构在重新分配宽度。
- 收起后只剩锚点按钮，按钮位置在展开态与收起态之间保持稳定，不做明显跳位。
- 工具条到内容区的过渡优先使用同色系边缘、浅内阴影和柔和明暗变化，不接受突兀黑线。
- 悬浮 `返回顶部` 按钮只在右区滚动后出现，固定在右下角覆盖层里，不再提供“滚到底部”按钮。

## 滚动模型

- 桌面端由页面壳层锁定整页滚动：`html/body` 与 `site-main` 不再承担主内容滚动。
- 主滚动发生在工作台内部：
  - 有左栏页面：左侧筛选与右侧内容各自独立滚动；
  - 无左栏页面：只有右侧主内容滚动。
- 筛选页右区继续承担：
  - 筛选后回到摘要区顶部；
  - 随机排序不强制回顶；
  - 从详情返回时恢复 query 和右区滚动位置。
- 非筛选页统一使用右区悬浮回顶入口，不操作 `window.scrollY`。

## 页面接入边界

- 页面只负责把内容映射到 `toolbarLead / toolbarPrimary / toolbarActions / sidebarHeader / sidebar / contentHeader / children`。
- 壳层本身统一负责：桌面锁滚、抽屉持久化、左右内滚容器、悬浮回顶覆盖层和 sidebarless 行为。
- 若后续页面暂时没有左侧内容，也只能走“隐藏左栏”模式，不能重新发明另一套页面外壳。
